# 🔢 Digit Recognition App (YOLO + CNN)

Приложение для автоматического распознавания числовых значений на изображениях и последующего переименования файлов.

Проект реализует полный ML pipeline: от детекции объекта до пользовательского интерфейса.

---

## 📌 Возможности

- Обработка папки с изображениями (`.jpg / .jpeg`)
- Автоматическое распознавание чисел
- Оценка уверенности модели:
  - **отлично** — высокая уверенность  
  - **!не уверена** — средняя уверенность  
  - **!пересмотр** — требуется проверка  
- Переименование файлов на основе предсказаний
- Работа с копией папки (оригиналы не изменяются)

---

## 🧠 Как работает pipeline

1. YOLO определяет объект и ключевые точки  
2. Выполняется перспективное преобразование (warp)  
3. Изображение подготавливается для OCR  
4. Сегментируются отдельные цифры  
5. Каждая цифра классифицируется CNN  
6. Формируется итоговое число и confidence

## 📈 Интерпретация

Система демонстрирует высокую точность среди уверенных предсказаний (~95%).

Статусы **"!не уверена"** и **"!пересмотр"** используются как механизм контроля качества и снижения количества ошибок.

## 🧠 Вывод

Ошибки связаны с совокупностью этапов:

**детекция → выравнивание → сегментация → классификация**

Основной источник ошибок — этап сегментации, а не классификация.

CNN модель достигает высокой точности на отдельных цифрах, однако ошибки возникают из-за:

- некорректного разделения цифр  
- потери части цифры при выравнивании  
- слияния соседних символов  

Таким образом, качество системы определяется устойчивостью всего pipeline.

## 🧪 Эксперименты

Были протестированы:

- Базовая CNN (10 классов)
- CNN с BatchNorm
- Добавление класса `not_digit`
- Усложнение pipeline (фильтры, scoring)

---

## 📉 Результаты

- Простая CNN показала лучшую стабильность  
- Усложнение модели не улучшило качество  

---

## 💡 Вывод

Эксперименты показали, что увеличение сложности модели и pipeline не всегда приводит к улучшению качества, особенно при ограниченном объеме данных.

---

## ⚠️ Ограничения

- Модель обучена только на цифрах (0–9)
- Чувствительна к качеству изображений
- Сегментация — ключевое узкое место
